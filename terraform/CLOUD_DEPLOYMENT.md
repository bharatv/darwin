# Darwin ML Platform - AWS Cloud Deployment (Terraform)

## Project Context

Darwin Distribution is an ML Platform deployment orchestration project that builds Docker images from service submodules and deploys the entire platform to Kubernetes using Helm charts. This document outlines the Terraform infrastructure required to deploy Darwin to AWS.

---

## Current Local Architecture

The platform currently runs on a local Kind cluster with the following setup:

| Component | Local Implementation |
|-----------|---------------------|
| Kubernetes | Kind cluster (1 control-plane + 2 workers) |
| Container Registry | Local registry at `localhost:5000` |
| Storage | hostPath mounts to `./kind/shared-storage` |
| S3/Object Storage | LocalStack mock at port 4566 |
| MySQL | Containerized MySQL 8.0 |
| OpenSearch | Containerized OpenSearch 2.11.0 |
| Kafka | Containerized (optional, disabled by default) |
| Cassandra | Containerized (optional, disabled by default) |

**Key Config Files:**
- `config.env` - Contains `DOCKER_REGISTRY` and `KUBECONFIG`
- `services.yaml` - Defines all applications, datastores, and operators
- `.setup/enabled-services.yaml` - User-selected services (generated by init.sh)
- `helm/darwin/values.yaml` - Helm chart default values

---

## Target AWS Architecture

### Decision: Fully Managed Cloud Services

Replace all local/containerized infrastructure with AWS managed services:

| Component | AWS Service |
|-----------|-------------|
| Kubernetes | Amazon EKS |
| Container Registry | Amazon ECR |
| Object Storage | Amazon S3 |
| MySQL Database | Amazon RDS MySQL |
| Search Engine | Amazon OpenSearch Service |
| Secrets | AWS Secrets Manager |
| Load Balancer | AWS ALB (via ALB Ingress Controller) |

## Module Specifications

### 1. VPC Module (`modules/vpc/`)

**Purpose:** Create isolated network for all Darwin resources.

**Resources:**
- VPC with CIDR `10.0.0.0/16`
- 3 Public subnets (10.0.1.0/24, 10.0.2.0/24, 10.0.3.0/24) - for ALB, NAT Gateway
- 3 Private subnets (10.0.11.0/24, 10.0.12.0/24, 10.0.13.0/24) - for EKS, RDS, OpenSearch
- Internet Gateway
- NAT Gateway (single for dev, multi-AZ for prod)
- Route tables
- VPC Endpoints for: ECR (api + dkr), S3, Secrets Manager, STS

**Outputs:** `vpc_id`, `private_subnet_ids`, `public_subnet_ids`

---

### 2. EKS Module (`modules/eks/`)

**Purpose:** Managed Kubernetes cluster for Darwin workloads.

**Resources:**
- EKS Cluster (Kubernetes 1.29)
- OIDC Provider (for IRSA - IAM Roles for Service Accounts)
- Managed Node Groups:
  - `general`: t3.large (2-5 nodes) - general workloads
  - `compute`: m5.xlarge (0-10 nodes) - ML compute (optional GPU: g4dn.xlarge)
- EKS Add-ons:
  - VPC CNI
  - CoreDNS
  - kube-proxy
  - EBS CSI Driver
  - AWS Load Balancer Controller
- IAM Roles:
  - Cluster role
  - Node role
  - IRSA roles for S3, Secrets Manager access

**Outputs:** `cluster_name`, `cluster_endpoint`, `cluster_ca_certificate`, `oidc_provider_arn`

---

### 3. ECR Module (`modules/ecr/`)

**Purpose:** Private container registry for Darwin images.

**Repositories to create:**
```hcl
repositories = [
  "darwin-mlflow",
  "darwin-mlflow-app",
  "darwin-compute",
  "darwin-cluster-manager",
  "ml-serve-app",
  "artifact-builder",
  "darwin-catalog",
  "darwin-ofs-v2",
  "darwin-ofs-v2-admin",
  "darwin-ofs-v2-consumer",
  "chronos",
  "darwin-workspace",
  "ray",
  "serve-md-runtime"
]
```

**Features:**
- Image scanning on push
- Lifecycle policy (keep last 10 images, delete untagged after 7 days)
- Encryption at rest (AWS managed key)

**Outputs:** `repository_urls` (map of name -> URL)

---

### 4. RDS Module (`modules/rds/`)

**Purpose:** Managed MySQL database replacing containerized MySQL.

**Resources:**
- RDS MySQL 8.0 instance
- DB Subnet Group (private subnets)
- Security Group (allow 3306 from EKS nodes only)
- Parameter Group (utf8mb4 charset)
- Secrets Manager secret for credentials

**Configuration:**

| Setting | Dev | Prod |
|---------|-----|------|
| Instance class | db.t3.medium | db.r5.large |
| Storage | 20GB gp3 | 100GB gp3 |
| Multi-AZ | No | Yes |
| Backup retention | 7 days | 30 days |

**Databases to create:**
- `darwin` (main database)
- `darwin_mlflow`
- `darwin_catalog`

**Outputs:** `endpoint`, `port`, `secret_arn`

---

### 5. S3 Module (`modules/s3/`)

**Purpose:** Object storage replacing LocalStack S3 mock.

**Buckets:**

| Bucket | Purpose | Versioning |
|--------|---------|------------|
| `{env}-darwin-mlflow-artifacts` | MLflow model artifacts | Yes |
| `{env}-darwin-shared-storage` | Shared platform storage | Yes |
| `{env}-darwin-terraform-state` | Terraform state (if not using existing) | Yes |

**Features:**
- Server-side encryption (AES-256)
- Lifecycle rules (transition to IA after 90 days, Glacier after 365 days)
- Block public access
- CORS configuration for MLflow UI

**Outputs:** `bucket_names`, `bucket_arns`

---

### 6. OpenSearch Module (`modules/opensearch/`)

**Purpose:** Managed search engine replacing containerized OpenSearch.

**Resources:**
- OpenSearch domain (v2.11)
- Deployed in private subnets
- Security Group (allow 443/9200 from EKS)
- Fine-grained access control enabled
- Master user credentials in Secrets Manager

**Configuration:**

| Setting | Dev | Prod |
|---------|-----|------|
| Instance type | t3.small.search | r5.large.search |
| Instance count | 1 | 3 |
| EBS volume | 20GB | 100GB |
| Zone awareness | No | Yes |

**Outputs:** `endpoint`, `dashboard_endpoint`, `secret_arn`

## Outputs for Helm Integration

**The Terraform outputs should generate a Helm values file:**

```yaml
# helm/darwin/cloud-values-aws.yaml (generated from Terraform outputs)
global:
  imageRegistry: <ecr_registry_url>
  storageClass: gp3
  namespace: darwin
  local-k8s: false

  database:
    mysql:
      host: <rds_endpoint>
      port: 3306
      database: darwin
      # Credentials from Secrets Manager via External Secrets Operator
      useExternalSecret: true
      secretName: darwin/rds-credentials

aws:
  region: <aws_region>
  s3:
    mlflowBucket: <mlflow_bucket_name>
    sharedBucket: <shared_bucket_name>
  opensearch:
    endpoint: <opensearch_endpoint>
    useExternalSecret: true
    secretName: darwin/opensearch-credentials

# Disable in-cluster datastores (using managed services)
datastores:
  enabled: false
  mysql:
    enabled: false
  localstack:
    enabled: false
  opensearch:
    enabled: false

services:
  enabled: true
```

## Implementation Order

1. **versions.tf** - Provider configuration
2. **modules/vpc/** - Network foundation
3. **modules/eks/** - Kubernetes cluster (depends on VPC)
4. **modules/ecr/** - Container registry (independent)
5. **modules/rds/** - Database (depends on VPC)
6. **modules/s3/** - Object storage (independent)
7. **modules/opensearch/** - Search engine (depends on VPC)
8. **main.tf** - Compose all modules
9. **variables.tf / outputs.tf** - Inputs and outputs
10. **environments/dev/** - Dev environment configuration
11. **cloud-values-aws.yaml** - Helm values template

## Post-Terraform Steps

After `terraform apply`, the following manual/scripted steps are needed:

1. **Update kubeconfig:**
   ```bash
   aws eks update-kubeconfig --name darwin-dev --region us-east-1
   ```

2. **Update config.env:**
   ```bash
   DOCKER_REGISTRY=<ecr_registry_url>
   KUBECONFIG=~/.kube/config
   ```

3. **Build and push images to ECR:**
   ```bash
   aws ecr get-login-password | docker login --username AWS --password-stdin <ecr_registry_url>
   ./setup.sh  # Modified to push to ECR
   ```

4. **Install External Secrets Operator** (for Secrets Manager integration):
   ```bash
   helm repo add external-secrets https://charts.external-secrets.io
   helm install external-secrets external-secrets/external-secrets -n external-secrets --create-namespace
   ```

5. **Deploy Darwin with cloud values:**
   ```bash
   helm upgrade --install darwin ./helm/darwin \
     --namespace darwin \
     --create-namespace \
     -f ./helm/darwin/cloud-values-aws.yaml
   ```

## Notes

- The Terraform uses modular design for reusability across environments
- All secrets are stored in AWS Secrets Manager, not in Terraform state
- EKS uses IRSA (IAM Roles for Service Accounts) for pod-level AWS access
- VPC endpoints reduce NAT Gateway costs for AWS service traffic
- OpenSearch is deployed in private subnets with no public access

## Local Testing Guide

This section describes how to test Terraform scripts locally without incurring AWS costs.

### Testing Options Overview

| Method | What It Tests | Cost |
|--------|---------------|------|
| `terraform validate` | Syntax and configuration | Free |
| `terraform plan` | Resource planning (dry run) | Free |
| LocalStack + tflocal | S3, ECR, IAM, Secrets Manager | Free |
| AWS Free Tier | Full infrastructure (minimal) | ~$2-5/day |

---

### Option 1: LocalStack + tflocal (Recommended)

Darwin already includes LocalStack in its stack. Use it to emulate AWS services locally.

**Install tflocal:**
```bash
pip install terraform-local
```

**LocalStack Service Support:**

| Service | LocalStack Support | Notes |
|---------|-------------------|-------|
| S3 | Full | Fully functional |
| ECR | Full | Fully functional |
| IAM | Full | Policies and roles |
| Secrets Manager | Full | Fully functional |
| RDS | Limited | Creates mock endpoint, no actual MySQL |
| EKS | Limited | Pro feature only, use Kind instead |
| OpenSearch | Limited | Pro feature only |

**Usage:**
```bash
# Instead of 'terraform' commands, use 'tflocal':
tflocal init
tflocal plan
tflocal apply
```

---

### Option 2: Terraform Validation Only

Test syntax and configuration without creating any resources:

```bash
# Validate HCL syntax
terraform validate

# Preview what would be created (dry run)
terraform plan

# Plan with specific variables
terraform plan -var="environment=test" -var="aws_region=us-east-1"
```

---

### Option 3: AWS Free Tier (Minimal Resources)

Create a `test.tfvars` with minimal resource sizes for cheap testing:

```hcl
# environments/test/terraform.tfvars
environment = "test"
aws_region  = "us-east-1"

# Use smallest possible instances
rds_instance_class       = "db.t3.micro"      # Free tier eligible
opensearch_instance_type = "t3.small.search"  # ~$0.036/hour
eks_node_instance_type   = "t3.micro"         # Free tier eligible

# Minimal node counts
eks_min_nodes = 1
eks_max_nodes = 1

# Skip expensive features
rds_multi_az         = false
create_nat_gateway   = false  # Use public subnets only for test
enable_vpc_endpoints = false
```

### Local Testing Workflow

**Step 1: Start LocalStack**
```bash
# Option A: Use Darwin's existing LocalStack
cd /path/to/darwin
docker-compose up -d localstack

# Option B: Start standalone LocalStack
docker run -d -p 4566:4566 --name localstack localstack/localstack
```

**Step 2: Test Supported Modules**
```bash
cd terraform/environments/local

# Initialize Terraform with LocalStack
tflocal init

# Test S3 module
tflocal apply -target=module.s3

# Test ECR module
tflocal apply -target=module.ecr

# Test Secrets Manager (part of other modules)
tflocal apply -target=module.secrets
```

**Step 3: Validate Unsupported Modules**
```bash
# These can't be fully tested locally, but we can validate the configuration
tflocal plan -target=module.vpc
tflocal plan -target=module.eks
tflocal plan -target=module.rds
tflocal plan -target=module.opensearch
```

**Step 4: Verify Resources in LocalStack**
```bash
# Check S3 buckets
aws --endpoint-url=http://localhost:4566 s3 ls

# Check ECR repositories
aws --endpoint-url=http://localhost:4566 ecr describe-repositories

# Check Secrets Manager
aws --endpoint-url=http://localhost:4566 secretsmanager list-secrets

# Check IAM roles
aws --endpoint-url=http://localhost:4566 iam list-roles
```

**Step 5: Clean Up**
```bash
tflocal destroy
docker stop localstack
```

### CI/CD Integration

For automated testing in CI/CD pipelines:


### Troubleshooting Local Testing

**Issue: LocalStack connection refused**
```bash
# Check if LocalStack is running
docker ps | grep localstack

# Check LocalStack logs
docker logs localstack

# Verify endpoint is accessible
curl http://localhost:4566/_localstack/health
```

**Issue: tflocal not found**
```bash
# Ensure pip packages are in PATH
export PATH=$PATH:~/.local/bin

# Or install globally
sudo pip install terraform-local
```

**Issue: Module depends on unsupported service**
```bash
# Use -target to test only supported modules
tflocal apply -target=module.s3 -target=module.ecr

# Or create a local-only main.tf that excludes unsupported modules
```

**Issue: State file conflicts**
```bash
# Use separate state for local testing
tflocal init -reconfigure
# Or delete existing state
rm -rf .terraform terraform.tfstate*
```
